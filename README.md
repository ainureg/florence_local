# Florence Home API

Лёгкий **локальный API для работы с моделью Florence** через Docker.

Основная цель проекта — **запустить Florence дома**, даже если модель **не собирается локально из-за зависимостей** (например `flash-attn`).
Контейнер изолирует окружение и предоставляет простой HTTP-интерфейс.

⚠️ **Проект предназначен для домашнего использования и экспериментов**, а не для production-нагрузки.

---

# Основная идея

Модель Florence запускается **внутри Docker**, а с хоста можно отправлять изображения через HTTP.

```
Python script (host)
        │
        │  POST /process
        ▼
Docker container
(FastAPI + Florence)
        │
        ▼
результат обработки
```

Это позволяет:

* запускать Florence даже если она **не собирается на основной системе**
* вызывать модель из **любых локальных Python-скриптов**
* легко интегрировать Florence в собственные инструменты

# Требования

* Docker
* NVIDIA GPU
* NVIDIA Container Toolkit

Проверка:

```bash
docker run --gpus all nvidia/cuda:12.1.1-base-ubuntu22.04 nvidia-smi
```

---

# Сборка контейнера

```bash
docker build -t florence-home .
```

---

# Запуск

```bash
docker run --gpus all -p 8000:8000 florence-home
```

После запуска API будет доступен:

```
http://localhost:8000
```

---

# Использование из Python

Пример отправки изображения:

```python
import requests

url = "http://localhost:8000/process/"

with open("image.jpg", "rb") as f:
    files = {"file": f}
    r = requests.post(url, files=files)

print(r.json())
```

---

# Пример запроса через curl

```bash
curl -X POST \
  -F "file=@image.jpg" \
  http://localhost:8000/process/
```

# Когда это полезно

Этот проект особенно удобен если:

* Florence **не собирается локально**
* возникают ошибки `flash-attn`
* хочется **изолировать зависимости**
* нужно быстро добавить Florence в локальные инструменты
